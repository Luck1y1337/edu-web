## 🎯 1. Суть и масштаб проекта

**O'quv Markaz** — это полнофункциональная образовательная платформа (EdTech / LMS) на узбекском языке. 
Проект имеет сложную структуру, разделенную на две основные части:

2. **Dashboard (Личный кабинет студента):** Обучение (уроки), Управление профилем, Сертификаты, Результаты, История платежей, Настройки.

**Текущий статус:** Все 24+ страницы уже сверстаны и связаны логикой. Базовая архитектура заморожена и стабильна.

## 🛠 2. Стек технологий (Tech Stack)

Проект использует самые современные инструменты 2024-2026 годов. Вы обязаны следовать этому стеку:
- **Ядро:** React 19 + TypeScript + Vite 8
- **Стилизация:** TailwindCSS v4 (используются утилитарные классы, кастомные цвета настроены).
- **Роутинг:** `react-router-dom` v7 (используются `Outlet`, `NavLink`, вложенные маршруты).
- **Глобальный Стейт:** `Zustand` (максимально легковесное управление состоянием без Redux).
- **Работа с API:** `@tanstack/react-query` (для кэширования и запросов) + `axios`.
- **Формы:** `react-hook-form` (для валидации логина, регистрации, настроек профиля).
- **Уведомления:** `react-toastify`.

## 🧠 4. Как работает Data Flow (Умный API Fallback)

Это самая важная часть архитектуры, которую вы **не должны сломать**.
Проект спроектирован так, чтобы работать **даже если бэкенд выключен или выдает ошибки (404/500)**.

**Как идет запрос данных:**
1. Страница вызывает кастомный хук: `const { data } = useCourses();`
2. Хук делает запрос через `React Query` к функции из `api.ts`.
3. `api.ts` делает реальный `axios.get(Endpoints.courses.list)`.
4. **ЕСЛИ УСПЕХ:** Данные с бэкенда проходят через `mappers.ts` (приводятся к типам фронтенда) и отдаются компоненту.
5. **ЕСЛИ ОШИБКА БЭКЕНДА:** `catch` блок перехватывает ошибку, логирует её, и **возвращает статические данные из папки `src/data/`**!
   
*Почему так сделано?* Чтобы верстка не разваливалась и проект можно было демонстрировать заказчику или тестировать UI без запущенного сервера. 
*Ваше правило:* Создавая новые хуки/запросы, всегда добавляйте fallback на статические данные.

## 🔐 5. Авторизация и Глобальный Стейт (Zustand)

Файл `src/store/user.store.ts` управляет сессией:
- Содержит `user` (объект с данными), `isAuthenticated` (boolean).
- Содержит функцию `logout()`, которая очищает стейт.
- **Хак с модалками:** В сторе лежит состояние `isLogoutModalOpen`. Это позволяет вызывать всплывающее окно выхода из ЛЮБОЙ точки приложения (будь то мобильное меню в Header или кнопка в Sidebar), не прокидывая пропсы через 10 слоев.

## 🎨 6. Дизайн-Система и TailwindCSS

Если вы меняете UI, сохраняйте премиальный вид. Проект использует строгую палитру:
- **Основной цвет (Primary):** Royal Blue (`text-blue-600`, `bg-blue-600` — `#2563EB`).
- **Тёмные акценты:** Persian Blue (`#1D4ED8`), Violet (`#6D28D9`).
- **Серые оттенки (Текст и Рамки):** `text-gray-900` (заголовки), `text-gray-500` (подзаголовки), `border-gray-100/200` (мягкие разделители), `bg-gray-50` (фоны страниц ЛК).
- **Шрифты:** 
  - `font-manrope` — для крупных заголовков (H1-H3). Дает премиальный, уверенный вид.
  - `font-inter` (по умолчанию) — для читаемого текста.
- **Градиенты:** Используются мягкие фоновые градиенты (`bg-gradient-to-r from-blue-50 to-purple-50`), избегайте "кислотных" резких переходов.
- **Скругления и Тени:** Используем `rounded-xl` или `rounded-2xl` для карточек, и мягкие тени `shadow-sm` или `shadow-md`. Никаких угловатых (`rounded-none`) блоков, если это не обосновано дизайном.

## ⚡ 7. Правила работы ИИ-Агента (STRICT RULES)

1. **Не меняй работающий код без прямого приказа.** Если пользователь просит добавить кнопку на страницу `X`, добавляйте кнопку только на `X`. Не трогайте `Y`.
2. **Всегда проверяй роуты.** Если создаешь новую страницу, не забудь зарегистрировать её в `src/routes/routes.tsx`.
3. **Нет "AnyScript".** Проект использует строгий TypeScript. Пишите интерфейсы в папке `src/types/`, типизируйте все `props`.
4. **Убирай за собой.** В проекте включены строгие линтеры. Никаких `TS6133: 'X' is declared but its value is never read`. Удаляйте неиспользуемые импорты.
5. **Абсолютные пути vs Относительные.** Обращайте внимание на то, где находится компонент.
6. **Адаптивность (Mobile-First).** Если верстаете новый блок, он ОБЯЗАН адекватно выглядеть на мобилках (`flex-col`, `lg:flex-row`, скрытие элементов на малых экранах).
7. **Ссылки.** НИКОГДА не используйте тег `<a href="...">` для внутренней навигации. ТОЛЬКО `<Link to="...">` или `<NavLink>` из `react-router-dom`. Иначе ломается SPA и страница перезагружается.
